Getting Started

Starting the Application

FamilyTree Social runs locally on your computer. No internet connection or cloud account is needed beyond your local network.

🪟 Windows

  1. Double-click start.bat in the project folder.
  2. If Node.js is not installed, the script installs it automatically via winget.
  3. A browser window opens at http://localhost:3000.
  4. Keep the command prompt open while using the app.

🐧 Linux / 🍎 Mac

  1. Open a terminal in the project folder.
  2. Run ./start.sh (or chmod +x start.sh && ./start.sh the first time).
  3. Node.js installs automatically via apt, brew, etc. if needed.
  4. Browser opens at http://localhost:3000.
💡
Port conflict? Run PORT=3001 ./start.sh (Linux/Mac) or edit start.bat and replace 3000 with any free port number.

Dashboard (Home)

The Dashboard is the first thing you see after logging in. It shows a quick overview of your trees and a combined activity feed from everyone across all your trees.

Tree Quick-Access Bar

Horizontally scrollable cards at the top show each tree you own or collaborate on. Click a card to jump straight into that tree.

Compose a Post

Share a message with your family directly from the Dashboard:

  1. Click inside the text area that says "Share a memory or update…"
  2. Type your message.
  3. Use the Select tree… dropdown to choose which tree the post belongs to.
  4. Choose a post type: Update, Memory, or Milestone.
  5. Click Post.

Activity Feed

Posts from all collaborators across all your trees appear here in reverse chronological order. Each post card shows:

  • Author avatar and name
  • Tree name (gold text) and post type badge
  • Post content
  • ❤️ Heart reaction button — click to like/unlike; the count updates immediately
  • 💬 Comment button — click to expand the comment thread
  • ✕ Delete button — only visible on your own posts
💡
Load more — if there are more posts than shown, a "Load More" button appears at the bottom of the feed.
4
Family Trees

Navigate to 🌳 Family Trees in the sidebar to see all your trees and ones you've been invited to collaborate on.

Tree Cards

Each tree is displayed as a card showing:

  • Cover photo (or a default 🌳 icon)
  • Tree name
  • Member count and collaborator count
  • Description snippet (first 80 characters)
  • Privacy badge: family private public
  • Your role badge: owner editor viewer

Click any card to open that tree.

Creating a New Tree

  1. Click + New Tree (top-right of the Trees page, or in the sidebar).
  2. Enter a Tree Name (required).
  3. Add an optional Description.
  4. Set the Privacy level.
  5. Click Create Tree. You're taken straight into the new tree.
Privacy LevelWho can see it
family FamilyOnly people you explicitly invite.
private PrivateOnly you (the owner).
public PublicAnyone with the link (read-only for non-members).

🌳 Tree View Tab — Visualization

The interactive SVG canvas fills the full browser width. The tree is arranged top-to-bottom: oldest ancestors at the top, youngest generations at the bottom.

Generation Bands & Colors

Each generation row has a subtle tinted background to make depth immediately readable. Edge (line) colors also cycle through a seven-color palette by generation:

2
3

Generation 0 (oldest) → violet, Gen 1 → blue, Gen 2 → teal … cycling every 7 generations.

Lineage Lines (S-Curves)

All parent → child connections are drawn as smooth S-shaped cubic bezier curves — no sharp corners or horizontal spines. When both parents are known, lines from each parent converge to a midpoint junction before flowing down to the child.

Panning & Zooming

ActionHow

FullscreenClick ⛶ Fullscreen; press Esc or click ✕ to exit.

Clicking a Node

Click any person's card in the tree to open their Member Detail panel, which shows their photo, lifespan, gender, birthplace, biography, and all family relationships (father, mother, spouse, children). Clicking a relative's name in that panel jumps to their detail.

Collapse / Expand Subtrees

Each node has a small ⊖ button on its bottom edge. Clicking it collapses all descendants of that person, hiding them from view. The collapsed node shows a ▶N badge indicating how many hidden members are below. Click ▶ to expand them again.

Lineage Filter

Use the three toggle buttons in the toolbar to switch between:

  • All — show the complete tree.
  • Paternal — show only the paternal (father's side) lineage from the focal person.
  • Maternal — show only the maternal (mother's side) lineage.

Use the View from ▾ dropdown to pick any member as the focal person for the lineage filter.

Search

  1. Click the 🔍 Search… box in the toolbar (or use the global search bar in the header).
  2. Type a name. Results appear as you type.
  3. Click a result — the tree zooms in and centers on that person, showing them together with one generation above (their parents) and one generation below (their children).

👥 Members Tab

A scrollable alphabetical list of all members in the tree. Each row shows:

  • Profile photo or initials avatar (color-coded by gender: blue = male, pink = female, purple = other)
  • Full name
  • Birth year, death year, and birthplace
  • Gender color dot on the right

Click any row to open that person's Member Detail panel.

💡
The Members tab scrolls independently — even with 185+ members you can reach everyone without the page scrolling.

📰 Feed Tab

Shows posts belonging to this specific tree. You can compose a new post here using the compose area at the top, and read, react to, and comment on posts below it — all scoped to this tree only.

🤝 People Tab

Lists the tree owner and all collaborators with their roles. If you are the owner you can remove any collaborator from this tab.

A + Invite People button at the bottom opens the invitation dialog (see Section 8 — Collaboration).

Managing Members

Adding Members

Click + Add Member to open the bulk-add form. You can add up to 6 members at once in a single operation.

  1. Fill in the first member's details (only Full Name is required).
  2. Click + Add another member to append a second form entry (up to 6 total).
  3. Click the ✕ Remove button next to any entry to discard it.
  4. Click Save Member(s) — all members are created and the tree updates.

Available Fields (Add Form)

FieldNotes
Full Name ★Required for the first entry.
GenderMale / Female / Other. Affects node border color in the tree.
Birth Year4-digit year, e.g. 1985.
Death YearLeave blank if the person is still alive.
Birth PlaceFree text, e.g. "Dublin, Ireland".
Father (Paternal Parent)Pick from existing members.
Mother (Maternal Parent)Pick from existing members.
Spouse / Partner (Primary)Pick from existing members. Sets the main partnership link.
BiographyOptional free-text biography.
📝
The Father/Mother/Spouse dropdowns only appear once there is at least one existing member in the tree to choose from.

Editing a Member

Open a member's detail panel (click them in the tree or Members tab), then click Edit. The edit form has all the same fields as the add form, plus two additional partner fields.

Additional Fields in Edit Form

FieldNotes
Spouse / Partner (Primary)The first or main partner — shown as the primary connection in the tree.
Additional PartnersMulti-select list for 2nd, 3rd, or more partners (see below).

Multiple Partners / Spouses

FamilyTree Social supports any number of partners per person. Partners are displayed side by side in the tree, connected by a dashed pink line with a ♥ heart at the midpoint.

How to Set Up Multiple Partners

  1. Open a member's edit form.
  2. Set the Spouse / Partner (Primary) dropdown to the first (or main) partner.
  3. In the Additional Partners multi-select, hold Ctrl (Windows/Linux) or ⌘ Cmd (Mac) and click each additional partner to select them.
  4. Click Save Member.
📝
Children are automatically linked to the correct couple. If a child has both a paternal and maternal parent set, the system infers the partnership and draws the connection — even if you haven't explicitly set the spouse/partner fields.

Member Photos

You can upload a profile photo for any member:

  1. Open the member's detail panel.
  2. Click 📷 Photo.
  3. Choose a JPG or PNG image (up to 5 MB).
  4. The photo appears in their node in the tree and on their detail panel immediately.

Deleting a Member

Open the member's detail panel and click Delete. You will be asked to confirm. Deleting a member removes them from the tree and clears any parent/spouse links that referenced them. This action cannot be undone.

Social Features

Post Types

TypeSuggested use
📢 UpdateGeneral news or information about the family tree.
🧠 MemoryA personal memory or story about a family member.
🏆 MilestoneSignificant life events — births, graduations, anniversaries.

Reacting to Posts

Click the ❤️ button on any post to add a heart reaction. Click it again to remove it. The count updates instantly for you; other collaborators see the updated count on their next page load.

Comments

  1. Click 💬 on a post to expand the comment thread.
  2. Type in the input field at the bottom of the thread.
  3. Press Enter to submit the comment.

Comments appear immediately for you and are visible to all collaborators on the tree.

Deleting Posts

You can only delete your own posts. Click the ✕ button on the top-right of a post card and confirm.

Inviting Someone

  1. Click Invite on the tree header toolbar (or + Invite People in the People tab).
  2. Enter the person's username or email address.
  3. Choose their role (Editor or Viewer).
  4. Add an optional personal message.
  5. Click Send Invite. A shareable link is also displayed — copy it to send manually.

Accepting an Invitation

When you receive an invitation link (e.g. http://localhost:3000/invite/<token>), open it in your browser. You'll see:

  • Who invited you and which tree
  • What role you're being given
  • Any personal message from the inviter

Click Join Family Tree to accept, or Decline to reject. If you're not logged in you'll be prompted to sign in or create an account first — the invitation will be saved and applied automatically after you log in.

Removing a Collaborator

Tree owners can remove collaborators from the 🤝 People tab by clicking the Remove button next to their name. They immediately lose access to the tree.

Import & Export

Export as JSON

Click ↓ JSON on the tree toolbar. A .json file is downloaded containing all member data including names, dates, relationships, and partner links. This file can be re-imported into any tree.

JSON File Structure

{
  "version": "1.0",
  "exported_at": "2026-05-19T10:00:00.000Z",
  "tree": {
    "name": "The Johnson Family",
    "description": "...",
    "privacy": "family"
  },
  "members": [
    {
      "id": "uuid",
      "name": "John Smith",
      "gender": "male",
      "birth_year": "1968",
      "death_year": null,
      "birth_place": "Dublin, Ireland",
      "bio": "...",
      "paternal_parent_id": "uuid-of-father",
      "maternal_parent_id": "uuid-of-mother",
      "spouse_id": "uuid-of-spouse",
      "partner_ids": "[\"uuid1\",\"uuid2\"]"
    }
  ]
}

Import from JSON

  1. Click ↑ Import JSON on the tree toolbar (requires editor/owner role).
  2. Choose a .json file previously exported from this app.
  3. A preview lists the first 15 member names. Confirm the count is correct.
  4. Click Import N Member(s). All relationships are remapped to new IDs automatically.
📝
Import is additive — existing members in the target tree are not affected. Imported members are added alongside them.

Export as Interactive HTML

Click ↓ HTML to download a fully self-contained HTML file. It includes:

  • All member data embedded as JavaScript
  • An interactive SVG tree with zoom, pan, and lineage filter
  • A member detail side-panel
  • No internet connection or server required to view it

Share this file with family members who don't have accounts — they can open it in any browser and explore the tree.

Profile & Settings

Profile Page

Click 👤 Profile in the sidebar to see your public profile, which includes:

  • Your avatar and display name
  • Your username and biography
  • Stats: trees owned, trees you collaborate on, total members across all your trees
  • A grid of your owned trees

Click Edit Profile to jump to Settings.

Settings Page

Click ⚙️ Settings in the sidebar. Two sections are available:

Profile Section

FieldNotes
Change PhotoUpload a new avatar image (JPG/PNG, up to 5 MB). Appears in the header and on posts.
Display NameThe name shown to other collaborators. Can be changed at any time.
BioA short text description shown on your profile page.
UsernameRead-only. Cannot be changed after account creation.
EmailRead-only. Contact the server administrator to change.

Change Password Section

  1. Enter your Current Password.
  2. Enter the New Password (at least 6 characters).
  3. Click Update Password.

Tree Settings

Each tree has its own settings dialog, accessible via ⚙️ Settings on the tree header (owner only):

  • Rename the tree
  • Edit the description
  • Change the privacy level
  • Upload a cover photo (JPG/PNG, up to 5 MB)
  • Delete Tree — permanently deletes the tree and all its members, posts, and comments
⚠️
Tree deletion is permanent. Export a JSON backup before deleting a tree you might want to restore later.

Notifications

FamilyTree Social sends in-app notifications for key events. The 🔔 bell in the header shows a red badge when there are unread notifications.

Notification Types

EventWho gets notified
New member added to a treeAll collaborators and the owner (except the person who added them).
Invitation acceptedTree owner and collaborators.

Viewing Notifications

There are two ways to view notifications:

  • Dropdown — click the 🔔 bell to see the latest notifications without leaving the current page. Click Mark all read to clear the badge.
  • Full page — click 🔔 Notifications in the sidebar for the complete list. All notifications are automatically marked as read when you open this page.

Notifications are polled every 30 seconds automatically — the badge updates without needing to refresh the page.

Troubleshooting
ProblemFix
Page is blank / white screenMake sure you opened http://localhost:3000 in your browser — not the index.html file directly from the file system.
Server won't start — "Port already in use"Run PORT=3001 ./start.sh (Linux/Mac) or edit start.bat and change 3000 to another port.
Windows: "Installation complete — please restart"Node.js was just installed for the first time. Restart your computer once, then double-click start.bat again.
Linux: "permission denied" on first runRun chmod +x start.sh once, then ./start.sh.
Linux: prompted for sudo passwordEnter your system password — this installs Node.js system-wide once only.
Tree looks empty after loginCreate a new tree first via + New Tree, then add members.
Members overlap in the treeClick ⊖ Fit in the tree toolbar to re-fit the layout. If the issue persists after adding many members, try zooming out with the − button.
Search finds no resultsEnsure you're spelling the name as it was entered. Search is case-insensitive but matches the exact spelling stored.
Photo upload failsCheck the file is a JPG or PNG smaller than 5 MB.
Invite link says "Invitation Not Found"Invitations expire or may have already been used. Ask the tree owner to send a new invitation.
Import JSON fails with "does not look like a family tree"Only files exported from FamilyTree Social are supported. Make sure you're importing a *-family-tree.json file downloaded from this app.
💡
Data is stored locally. All family data is in a SQLite database at data/familytree.db inside the project folder. Back this file up regularly to protect your data.
